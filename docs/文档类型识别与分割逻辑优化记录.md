### 1. 文档解析层（FileStorageService.java）

#### 1.1 引入 Apache Tika 自动格式识别
**优化前问题：**
- 手动维护 switch-case 分支处理不同格式（txt/md/doc/docx/pdf）
- 新增格式需要修改代码，违反开闭原则
- 仅基于文件扩展名判断，容易被伪造格式欺骗
- 缺少 MIME 类型检测和内容特征识别

**优化后方案：**
```java
// 使用Tika自动识别文档类型并解析（底层：魔数校验+后缀初筛+内容特征匹配）
ApacheTikaDocumentParser tikaParser = new ApacheTikaDocumentParser();
Document document = tikaParser.parse(Files.newInputStream(file.toPath()));
```

**技术优势：**
1. **三层识别机制**：
   - 第一层：文件后缀名初筛
   - 第二层：文件魔数（二进制签名）校验，防止伪造格式
   - 第三层：内容特征二次匹配
2. **统一解析接口**：无需手动分支处理，新增格式零代码改动
3. **元数据自动注入**：自动提取 `mime_type`、编码、作者等元信息
4. **异常容错**：加密PDF、损坏Office文件会抛出明确异常

  ↓

  - 基于 MIME 类型检测最佳策略
  - 降级到扩展名判断

  ↓
DocumentSplitter 执行分割
  - SEMANTIC → 递归分割器（段落→行→句子→单词→字符）

  - HYBRID → 行分割器
  ↓
List<TextSegment> 返回分片
  - 继承原文档全部元数据
